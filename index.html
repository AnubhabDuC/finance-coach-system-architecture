Distribution of Core Technologies

LLM-First Architecture

Unlike traditional systems, the core data extraction logic relies on Structured LLM Outputs (JSON Schema) rather than regex. This supports varied formats from Gmail, Android, and now iOS.

Mobile Integration (Phase 1.5)

Android: Listens for background notifications.
iOS: Uses Share Extensions to push PDFs/HTML from Safari or Files directly to the /mobile/ingest endpoint.

Core Data Pipelines

Understanding how raw data is transformed into structured financial insights.

1

Ingestion & Routing

Raw inputs (PDFs, HTML emails, SMS) arrive via Mobile API or Gmail API. They are hashed for deduplication and stored raw in MinIO.

2

Intelligent Extraction

The worker picks up the job. The LLM Extractor converts unstructured text into a strict JSON schema. Validators cross-check dates and amounts.

3

Normalization & Storage

Merchant names are normalized (e.g., "Uber *Ride" -> "Uber"). The clean transaction is written to PostgreSQL for analytics and the Agent.
